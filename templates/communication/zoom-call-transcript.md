# Zoom Call Transcript

Use this template when summarising a raw Whisper transcript for the `## Zoom Call Transcript` section of `notes.md` and the Web App "Zoom Call Transcript" tab.

## Audience

The summary is for a **technical/engineering audience**. An engineer will use this to investigate the issue — focus on technical facts, not conversational fluff.

## Required Sections

The summary **must** include exactly these three sections:

### Summary of Issue
Concise description (2-3 sentences) of the core problem the customer described on the call.

### Findings on Call
Key technical details discovered during the call:
- Configurations discussed
- Error messages or symptoms observed
- Diagnostic steps taken
- Environment details (versions, infrastructure, etc.)
- Any relevant timeline (when it started, what changed)

### Workaround
Any workaround discussed or agreed upon on the call. If none was discussed, write "None identified".

## Guidelines

- Be concise and technical — strip out small talk, pleasantries, and filler
- Preserve exact error messages, metric names, config values, and version numbers from the transcript
- If the transcript is unclear on a point, note it (e.g. "Customer mentioned a config change but specifics were unclear")
- Do not invent details that are not in the raw transcript
